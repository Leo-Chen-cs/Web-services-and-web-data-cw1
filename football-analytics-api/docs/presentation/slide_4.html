Modular Directory Structure
The project follows a modular architecture separating concerns into directories: app/routers/, app/models/, app/schemas/, app/services/, app/utils/.
Relational Database Schema
Core entities form the football model plus a User entity for authentication:
Team (Club details)
Player (FK to Team)
Match (FK to Home/Away Teams)
User (Authentication)
API Versioning & Middleware
Endpoints are prefixed with /api/v1/. Middleware includes CORS, timing (X-Process-Time) and global exception handling.
