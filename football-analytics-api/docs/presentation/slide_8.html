Key Challenges Overcome and Lessons Learned During Development

Technical Challenges
bcrypt Compatibility
Encountered a ValueError due to incompatibility between passlib and bcrypt>=4.1.0. Resolved by pinning dependency to 4.0.1.
Complex Aggregation
League table required processing 380 matches. Chose Python-based aggregation over complex SQL for better readability and maintainability.
Test Isolation
Tests initially interfered with each other due to shared state. Solved by implementing per-test database creation/teardown in fixtures.
Future Improvements
Database Migrations
Implement Alembic for safe, version-controlled database schema changes over time.
Redis Caching
Cache expensive analytics endpoints (like the league table) to significantly improve response times under heavy load.
Live Data Integration
Integrate with external APIs (e.g., Sportmonks) via background tasks to keep the database automatically updated.
Rate Limiting
Implement IP-based or token-based rate limiting to prevent abuse in a production environment.
